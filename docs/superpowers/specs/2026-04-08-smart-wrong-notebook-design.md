# Smart Wrong Notebook 设计文档

- 日期：2026-04-08
- 项目名：smart-wrong-notebook
- 目标平台：一期 Android APK，产品结构按未来 iOS 可扩展设计
- 产品定位：面向学生本人的 AI 错题本，支持拍照录题、OCR 识别、AI 解题、举一反三、错题沉淀与复习

## 1. 目标与范围

### 1.1 一期目标
一期需要从零做一个新的移动端产品，不保留旧项目实现形态，不使用 Docker，不做 Web 界面，不做复杂用户系统，先完成单机版闭环。

一期的核心闭环是：

拍照/导入 → 校正与框选 → OCR → AI 解题 → AI 举一反三 → 保存到错题本 → 后续复习

### 1.2 目标用户
主要用户是学生本人，产品应成为“他的个人错题本”，强调日常自主学习，而不是家长端陪伴工具。

### 1.3 一期必须支持的能力
- 拍照上传错题
- 从相册导入错题图片
- 图片校正与框选
- OCR 识别题目内容
- 用户确认或修正识别文本
- AI 解题与分步讲解
- AI 总结知识点、错因、学习建议
- AI 生成举一反三题目
- 变式题支持简单 / 同级 / 提高三档难度
- 本地存储错题、解析结果、练习记录
- 按分类查看和复习错题

### 1.4 一期明确不做
- Docker 部署
- Web 界面
- 复杂用户系统
- 云同步
- 手机号验证码登录
- 第三方题库接入
- 自动多题拆分
- 家长端、多用户协作、社交功能
- 复杂学习计划系统

### 1.5 参考策略
参考 wrong-notebook 及用户提供的朋友移植版，重点参考其能力链路与交互流程，不照抄原 UI。新项目采用新的移动端产品结构与新的视觉方案，并支持浅色 / 深色双主题。

## 2. 产品结构

一期采用 4 个主导航模块 + 1 个轻量设置区：

### 2.1 首页
目标是让学生快速开始录题或复习，而不是堆叠过多管理信息。

首页包含：
- 今日待复习数量
- 最近学习概览
- 主按钮：拍错题
- 次入口：相册导入
- 最近新增错题
- 最近生成的举一反三题

### 2.2 错题本
作为错题的主列表页，支持检索、筛选和回看。

支持：
- 学科筛选
- 标签筛选
- 掌握状态筛选
- 时间排序
- 关键词搜索

列表卡片展示：
- 题目缩略图或文本摘要
- 学科 / 标签
- 最近学习时间
- 掌握状态
- 是否有举一反三记录

### 2.3 复习
承接学习与巩固，而不是单纯浏览存量错题。

包含：
- 今日待复习
- 推荐复习队列
- 按未掌握 / 模糊 / 已掌握分组
- 重新做一遍入口
- 进入举一反三练习入口

### 2.4 我的 / 设置
保持轻量，但预留未来扩展入口。

包含：
- AI 模型配置
- OCR / 识别相关配置
- 数据导出 / 清理
- 账号与同步入口（仅预留）
- 关于产品

### 2.5 设置分层策略
产品分成两层：
- 主使用层：拍题、看解析、练习、复习、管理错题
- 高级设置层：AI 服务商、模型选择、Prompt 高级设置、数据导出

这样可以避免产品在一期变成开发者工具。

## 3. 核心流程设计

### 3.1 流程 A：录入错题
首页拍照 / 相册导入 → 校正与框选 → OCR 识别确认 → AI 分析中 → 结果页 → 保存到错题本

### 3.2 流程 B：学习错题
错题本列表 → 错题详情 → 查看解析 / 知识点 / 错因 → 生成举一反三 → 进入练习

### 3.3 流程 C：日常复习
首页或复习页进入 → 待复习列表 → 做题或看题 → 标记掌握状态 → 更新复习记录

### 3.4 AI 结果确认与保存策略
AI 解析结果页承担保存入口。用户先查看 OCR 和 AI 的结果是否合理，确认后再保存到错题本，而不是拍完立即入库。

这样可以减少错误 OCR 或错误解析污染错题库。

## 4. 拍照、校正与 OCR 设计

### 4.1 录题入口
一期支持两种入口：
- 拍照
- 相册导入

### 4.2 校正与框选能力
一期选择中配方案，支持：
- 手动框选题目区域
- 旋转图片
- 自动拉正 / 透视校正
- 进入 OCR 前的确认

不做自动多题拆分，但要在架构和交互上为未来扩展预留空间。

### 4.3 为什么不做自动多题拆分
自动多题拆分会引入新的交互、数据建模和失败场景：
- 一张图对应多道题
- 拆分错误后的回退与编辑
- 多个 OCR 和多个 AI 任务的关联

这会显著放大一期复杂度，因此不纳入一期。

### 4.4 OCR 结果编辑
一期要求 OCR 识别结果可由用户确认并修正。至少支持直接编辑题目文本、调整学科与标签，避免 OCR 错误导致后续 AI 解析失真。

## 5. AI 能力设计

### 5.1 AI 解题输出
对于每道错题，AI 需要输出：
- 题目理解结果
- 分步解题过程
- 最终答案
- 知识点总结
- 错因分析
- 学习建议

### 5.2 举一反三输出
一期采用增强方案，要求 AI 生成：
- 相似题
- 每道题的答案与解析
- 难度分层：简单 / 同级 / 提高

### 5.3 AI 提供方式
移动端本地保留错题数据，AI 能力走线上模型服务。产品应支持后续接入不同大模型服务商，但一期只需完成可配置、可测试连接、可实际调用。

## 6. 错题数据模型

### 6.1 基础信息
每道错题至少包含：
- 原始图片
- OCR 识别文本
- 用户修正后的题目文本（可选）
- 学科
- 标签
- 创建时间
- 最近学习时间

### 6.2 AI 内容
每道错题保存：
- AI 解题结果
- 分步讲解
- 知识点总结
- 错因分析
- 学习建议
- 举一反三题目列表
- 每道变式题的答案与解析
- 变式题难度等级

### 6.3 学习状态
每道错题保存学习状态：
- 未复习
- 复习中
- 已掌握

### 6.4 复习记录
保存：
- 复习次数
- 最近复习结果
- 历史掌握变化
- 是否收藏

## 7. 状态设计

### 7.1 内容状态
- processing：识别或 AI 分析中
- ready：结果可查看与复习
- failed：识别或 AI 处理失败

### 7.2 学习状态
- new：刚加入
- reviewing：持续复习中
- mastered：用户已掌握

### 7.3 举一反三练习状态
针对生成题：
- 未开始
- 已作答
- 答对
- 答错

## 8. 分类与组织方式

一期分类采用“双层策略”：

### 8.1 自动分类
- 学科
- AI / OCR 提取知识点
- 创建时间

### 8.2 用户可编辑分类
- 自定义标签
- 收藏
- 掌握状态

一期不做复杂知识图谱，只提供高频、可解释、可编辑的分类方式。

## 9. 页面设计原则

### 9.1 视觉策略
- 使用新的视觉方案，不照抄参考应用
- 支持浅色与深色两套主题
- 整体气质偏学生向、专注、现代、具有 AI 工具感

### 9.2 交互参考边界
会参考用户提供应用中的这些内容：
- 首页搜索与筛选组织方式
- 拍照后进入校正与框选步骤
- 设置页对 AI 服务商与学科管理的组织方式

不会直接照搬：
- 具体 UI 细节
- 全部视觉样式
- 过重的高级配置暴露方式

## 10. 异常处理

### 10.1 拍照 / 导入异常
场景：权限未授权、图片加载失败、图片尺寸异常、用户取消拍摄

处理要求：
- 提供明确提示
- 提供重新选择图片入口
- 不让流程卡死

### 10.2 校正 / OCR 异常
场景：框选不合适、OCR 结果差、学科识别错误

处理要求：
- 支持返回重新框选
- 支持重新识别
- 支持用户手动修正识别文本

### 10.3 AI 解析异常
场景：模型未配置、API key 无效、请求超时、返回不完整、题目不适合解析

处理要求：
- 区分配置问题与解析失败
- 提供重试能力
- 保留当前题目上下文，避免要求用户重拍

### 10.4 保存异常
场景：本地存储失败、图片保存失败、数据结构异常

处理要求：
- 尽量保留临时结果
- 支持再次保存
- 不直接丢失解析结果

### 10.5 举一反三异常
场景：未生成可用题目、难度不合理、重复度过高

处理要求：
- 支持重新生成
- 支持调整难度
- 不影响原错题入库和复习

## 11. 可扩展性要求

虽然一期只做单机 Android，但整体结构应为以下扩展预留位置：
- iOS 客户端
- 账号登录
- 云同步
- 第三方题库 API
- 多题自动拆分
- 更丰富的复习算法

预留意味着信息架构和数据边界不能写死为一次性 Demo，但实现上不提前引入重型复杂度。

## 12. 一期验收标准

只有下面这条闭环完整可用，一期才算成立：

拍照/相册导入 → 题目校正与框选（含旋转、自动拉正）→ OCR 识别与人工确认 → AI 解题 → AI 举一反三（简单 / 同级 / 提高）→ 保存到本地错题本 → 后续可按分类查看和复习

如果核心链路中任意一步不能稳定使用，就不应视为一期完成。
